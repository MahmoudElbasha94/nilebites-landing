4) MANDATORY MERGE CONFLICT (NAVBAR LINKS)
============================================================

Objective:
- Branch A adds one navigation link
- Branch B adds another navigation link
- Merging them causes a conflict
- You resolve it correctly

------------------------------------------------------------
A) Branch A
------------------------------------------------------------

git checkout -b feature/nav-link-reservations

In index.html, inside .nav-links, add:
<a href="#reservations">Reservations</a>

git add index.html
git commit -m "feat: add Reservations link to navbar (closes #2)"
git push -u origin feature/nav-link-reservations

Open PR but do NOT merge yet.

------------------------------------------------------------
B) Branch B
------------------------------------------------------------

git checkout main
git pull
git checkout -b feature/nav-link-offers

Add:
<a href="#offers">Offers</a>

git add index.html
git commit -m "feat: add Offers link to navbar (refs #2)"
git push -u origin feature/nav-link-offers

------------------------------------------------------------
C) Create and Resolve Conflict
------------------------------------------------------------

Merge Branch A first.

Then merge Branch B into main locally:

git checkout main
git pull
git merge origin/feature/nav-link-offers

You will see conflict markers:

<<<<<<< HEAD
...Reservations...
=======
...Offers...
>>>>>>> origin/feature/nav-link-offers

Resolve by keeping BOTH links:

<a href="#menu">Menu</a>
<a href="#about">About</a>
<a href="#offers">Offers</a>
<a href="#reservations">Reservations</a>
<a href="#contact">Contact</a>

Complete the merge:

git add index.html
git commit -m "merge: resolve navbar links conflict (offers + reservations)"
git push
